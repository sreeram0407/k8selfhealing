================================================================
1. ONE-PARAGRAPH PITCH
================================================================
This is an autonomous Kubernetes self-healing agent. A mock K8s
cluster emits failure events (CrashLoopBackOff, OOMKilled, etc.).
The agent, powered by Claude with tool use, investigates using
read-only MCP tools (get_pod_status, get_pod_logs, get_events,
get_deployment_info), diagnoses the root cause, then either
auto-remediates (restart_pod, rollback_deployment,
update_resource_limits, scale_deployment) or escalates to a
human via an "OpenClaw" integration. Every action is gated by a
guardrails layer (rate limits, blast radius, memory caps,
cooldowns) and every decision is written to a SQLite audit log.


================================================================
2. ARCHITECTURE
================================================================
    K8s Cluster (Mock)
           │
           ▼
      Event Watcher ──► AI Agent (Claude API) ──► MCP Tools ──► Auto-fix
                             │                                     OR
                             │                             Escalate (OpenClaw)
                             ▼
                        Audit Log (SQLite)

COMPONENTS
  mock_cluster.py          In-memory K8s simulator: pods, deployments,
                           events, failure injection, realistic log
                           templates per failure type.
  mcp_server.py            Tool definitions (Claude tool_use schema) +
                           MCPToolHandler that dispatches Claude's tool
                           calls to the cluster.
  agent.py                 The agentic loop: multi-turn tool_use with
                           Claude, guardrail gate, audit logging.
  guardrails.py            Safety checks (rate limits, blast radius,
                           memory caps, cooldowns, rollback window).
  openclaw_integration.py  Alert formatting / human-in-the-loop.
  audit.py                 SQLite-backed audit trail.
  config.py                Typed dataclass config + tiny YAML parser.
  mock_anthropic.py        Offline deterministic stand-in for the
                           Claude client so the demo runs without an
                           API key.

CONTROL FLOW (agent.handle_event)
  1. Receive event → build user message describing the alert.
  2. Claude messages.create with tools=TOOL_DEFINITIONS.
  3. Loop up to 10 turns:
       - If stop_reason == "end_turn" → done.
       - If stop_reason == "tool_use":
           * For each tool_use block:
             - If it's a remediation tool, gate through guardrails.check().
             - If BLOCKED: return a tool_result with GUARDRAIL_BLOCKED
               and an instruction to escalate. Claude sees this and
               calls alert_human on the next turn.
             - Else: execute via MCPToolHandler, record the action.
       - Append assistant response + tool_results to messages, loop.
  4. Write a row to audit_log.

"""AI Agent — core decision loop using the Claude API with tool use."""
from __future__ import annotations
import json, os, uuid
from typing import Any
try:
    import anthropic  # type: ignore
except ImportError:
    anthropic = None  # type: ignore

from .audit import AuditLogger
from .config import Config
from .guardrails import Guardrails
from .mcp_server import MCPToolHandler, TOOL_DEFINITIONS
from .mock_anthropic import MockAnthropicClient
from .mock_cluster import MockCluster
from .openclaw_integration import OpenClawIntegration

SYSTEM_PROMPT = """\
You are a senior Site Reliability Engineer (SRE) operating an autonomous
Kubernetes self-healing agent. Your job is to diagnose failing pods,
determine the root cause, and take the least-disruptive corrective action.

## Investigation Protocol
1. When notified of an unhealthy pod, ALWAYS investigate first:
   - get_pod_status to see current state
   - get_pod_logs to read recent logs
   - get_events to check related cluster events
   - get_deployment_info to check for recent deployments
2. Only after investigation, determine the root cause.

## Remediation Playbook
- CrashLoopBackOff + recent deployment   → rollback_deployment
- CrashLoopBackOff + NO recent deployment → restart_pod (if restart limit not hit)
- OOMKilled                               → update_resource_limits (+~50%)
- ImagePullBackOff                        → alert_human
- Pending pods                            → alert_human (capacity planning)
- >50% pods unhealthy                     → alert_human immediately
- High error rate across many pods        → scale_deployment

## Rules
- Prefer the LEAST DISRUPTIVE fix.
- If unsure, ESCALATE via alert_human.
- Never guess — investigate thoroughly first.
"""

_REMEDIATION_TOOLS = {
    "restart_pod", "scale_deployment",
    "rollback_deployment", "update_resource_limits",
}


class Agent:
    def __init__(self, config, cluster, audit, guardrails, openclaw):
        self.config = config
        self.cluster = cluster
        self.audit = audit
        self.guardrails = guardrails
        self.openclaw = openclaw
        self.client = self._build_client()
        self.tool_handler = MCPToolHandler(
            cluster,
            alert_callback=lambda inp: openclaw.format_alert_for_tool(inp),
        )

    @staticmethod
    def _build_client():
        use_mock = os.environ.get("DEMO_MOCK") == "1" or not os.environ.get("ANTHROPIC_API_KEY")
        if use_mock or anthropic is None:
            return MockAnthropicClient()
        return anthropic.Anthropic()

    def handle_event(self, event):
        event_id = str(uuid.uuid4())[:8]
        pod_name = event.get("pod_name", event.get("involved_object", ""))
        namespace = event.get("namespace", "default")
        user_msg = self._build_alert_message(event)
        diagnosis, action_taken, action_params, reasoning, tokens = self._run_agent_loop(
            user_msg, pod_name, namespace, event_id
        )
        audit_entry = {
            "event_id": event_id,
            "pod_name": pod_name,
            "namespace": namespace,
            "event_type": event.get("reason", ""),
            "diagnosis": diagnosis,
            "action_taken": action_taken,
            "action_params": action_params,
            "guardrail_check": "passed",
            "outcome": "success",
            "llm_reasoning": reasoning,
            "tokens_used": tokens,
        }
        self.audit.log(audit_entry)
        return audit_entry

    def _build_alert_message(self, event):
        pod_name = event.get("pod_name", event.get("involved_object", "unknown"))
        return (
            f"ALERT: Pod '{pod_name}' in namespace '{event.get('namespace', 'default')}' "
            f"has entered state '{event.get('status', event.get('reason', 'Unknown'))}'.\n"
            f"Event message: {event.get('message', 'N/A')}\n\n"
            f"Please investigate this pod, determine the root cause, and take "
            f"appropriate action."
        )

    def _run_agent_loop(self, user_message, pod_name, namespace, event_id):
        messages = [{"role": "user", "content": user_message}]
        reasoning_parts, total_tokens = [], 0
        action_taken, action_params, diagnosis = "none", {}, ""
        max_turns = 10

        for turn in range(max_turns):
            response = self.client.messages.create(
                model=self.config.agent.model,
                max_tokens=self.config.agent.max_tokens,
                system=SYSTEM_PROMPT,
                tools=TOOL_DEFINITIONS,
                messages=messages,
            )
            total_tokens += response.usage.input_tokens + response.usage.output_tokens

            for block in response.content:
                if block.type == "text" and block.text:
                    reasoning_parts.append(block.text)

            if response.stop_reason == "end_turn":
                diagnosis = "\n".join(reasoning_parts)
                break

            if response.stop_reason == "tool_use":
                tool_results = []
                for block in response.content:
                    if block.type != "tool_use":
                        continue
                    tool_name, tool_input = block.name, block.input

                    # Guardrail gate for remediation tools
                    if tool_name in _REMEDIATION_TOOLS:
                        allowed, reason = self.guardrails.check(tool_name, tool_input)
                        if not allowed:
                            tool_results.append({
                                "type": "tool_result",
                                "tool_use_id": block.id,
                                "content": json.dumps({
                                    "error": "GUARDRAIL_BLOCKED",
                                    "reason": reason,
                                    "instruction": (
                                        "This action was blocked by a safety guardrail. "
                                        "Please escalate to a human operator via alert_human."
                                    ),
                                }),
                            })
                            action_taken = f"blocked:{tool_name}"
                            action_params = tool_input
                            self.audit.log({
                                "event_id": event_id,
                                "pod_name": pod_name,
                                "namespace": namespace,
                                "action_taken": tool_name,
                                "action_params": tool_input,
                                "guardrail_check": reason,
                                "outcome": "guardrail_blocked",
                            })
                            continue

                    result_str = self.tool_handler.handle(tool_name, tool_input)
                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": result_str,
                    })

                    if tool_name in _REMEDIATION_TOOLS:
                        self.guardrails.record_action(tool_name, tool_input)
                        action_taken = tool_name
                        action_params = tool_input
                    if tool_name == "alert_human":
                        action_taken = "escalated"
                        action_params = tool_input

                messages.append({"role": "assistant", "content": response.content})
                messages.append({"role": "user", "content": tool_results})

        diagnosis = diagnosis or "\n".join(reasoning_parts) or "Agent completed without explicit diagnosis."
        return diagnosis, action_taken, action_params, "\n".join(reasoning_parts), total_tokens

6. FILE: src/guardrails.py (key logic)
================================================================
class Guardrails:
    """
    Checks in order (order matters — cheapest / most catastrophic first):
      1. Global rate limit (max_actions_per_hour)
      2. Blast radius (if >50% pods in namespace unhealthy → refuse)
      3. Per-pod cooldown
      4. Action-specific:
         - restart_pod:         max 3 restarts/hour per pod
         - scale_deployment:    0 <= replicas <= max_replicas
         - rollback_deployment: last deploy must be within rollback_window_minutes
         - update_resource_limits: new memory <= original * max_memory_multiplier

    Memory baseline is captured the FIRST time a pod is seen, so repeated
    bumps cannot compound past the multiplier.
    """

10. DESIGN DECISIONS I WANT TO TALK ABOUT
================================================================
- Guardrails live OUTSIDE the LLM, in plain code, because we cannot
  trust a model to self-enforce safety. The LLM proposes; a
  deterministic check disposes.
- When a guardrail blocks, we return the error back to Claude as a
  tool_result (not a silent drop). This lets the model adapt
  within-turn — typically by calling alert_human. It turns safety
  violations into signal rather than dead ends.
- Tools are split into READ (investigation) and WRITE (remediation).
  Only WRITE tools are gated. This mirrors real RBAC reasoning.
- Rate-limit windows are enforced via timestamp-prune on a deque-like
  list rather than a sliding counter, so bucket edges don't create
  false allows.
- Memory multiplier uses the ORIGINAL memory seen on first contact,
  not the most recent value, so the model can't climb 256→512→1024
  through repeated 2× bumps.
- Blast radius is computed per-namespace and checked before ANY
  remediation. Even a low-rate-limit action can't fire during an
  incident that's clearly systemic — the right move is "page a
  human," not "poke one pod."
- Audit log captures diagnosis + reasoning + tokens. This gives a
  postmortem artifact and cost trace for every decision.


================================================================
11. WEAK SPOTS AN INTERVIEWER MIGHT PROBE
================================================================
- The cluster is a mock. Real K8s has eventual consistency,
  reconciliation loops, admission webhooks — none of that is modeled.
- The guardrails state is in-process and resets on restart. In prod
  it'd need to live in Redis / a durable store so restarts don't
  reset rate limits.
- There's no retry/backoff on Claude API errors.
- The "MCP server" here is effectively just tool definitions; a real
  MCP server would be a separate process over stdio/HTTP.
- The mock Anthropic client is deterministic and can hide real-model
  failure modes (hallucinated tools, over-eager remediation, loops).
- No cost/token budget per event — max_turns is the only loop bound.
- audit_log has no indices and keeps llm_reasoning raw — would get
  large.
